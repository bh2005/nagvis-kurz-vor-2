NagVis 2 – API Dokumentation
REST + WebSocket · JWT Bearer Auth

Übersicht

Die NagVis-2-API ist eine REST/WebSocket-API auf Basis von FastAPI. Alle /api/*-Endpunkte erfordern einen JWT-Bearer-Token im Authorization-Header. Der WebSocket-Endpunkt akzeptiert den Token als Query-Parameter ?token=.

Base URL: http://<host>:8080
Interaktive Swagger-UI: /docs

Request-Format

# Alle /api/*-Endpunkte
Authorization: Bearer <jwt-token>
Content-Type: application/json

HTTP-Status-Codes

CodeBedeutung
200OK
201Erstellt
400Ungültige Anfrage (Validierungsfehler)
401Kein oder ungültiges Token
403Rolle nicht ausreichend
404Ressource nicht gefunden
409Konflikt (z.B. Map-ID existiert bereits)
413Datei zu groß

Fehler-Response

{
  "detail": "Fehlermeldung"
}

Auth-Flow

NagVis 2 verwendet stateless JWT-Bearer-Tokens (HS256). Tokens werden einmalig erzeugt und im Authorization-Header mitgeschickt.

1
Admin-Token erzeugen (einmalig, CLI)

Das erste Token wird direkt auf dem Server erzeugt:

2
Weitere Tokens über die API anlegen

Mit dem Admin-Token können weitere Tokens für andere User/Rollen erzeugt werden.

3
Token in jedem Request mitschicken

REST: Authorization: Bearer <token>
WebSocket: ws://host/ws/map/id?token=<token>

4
Token widerrufen (optional)

Über DELETE /api/auth/tokens/{jti} kann ein Token sofort gesperrt werden.

Schritt 1 – Erstes Admin-Token erzeugen
python3 -c "
from backend.core.auth import AuthManager
am = AuthManager()
print(am.create_token('admin', 'admin'))
"
Schritt 2 – Weitere Tokens über die API
curl -X POST http://localhost:8080/api/auth/tokens \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{"username": "alice", "role": "editor", "expires_in": 86400}'
Schritt 3 – Token verwenden
curl http://localhost:8080/api/maps \
  -H "Authorization: Bearer eyJhbGci..."
Der Secret Key muss als Umgebungsvariable NAGVIS_SECRET gesetzt sein. Ohne ihn wird ein unsicherer Dev-Key verwendet.

Rollen

Rollen sind kumulativ – admin hat alle editor-Rechte, editor alle viewer-Rechte.

RolleRechte
viewer Maps und Host-Status lesen, WebSocket verbinden
editor + Objekte anlegen/verschieben/löschen, Hintergrund hochladen, Maps erstellen/umbenennen
admin + Maps löschen, Backends abfragen, Tokens verwalten, NagVis-1-Migration

Auth

POST /api/auth/tokens Neues Token erzeugen admin ▾
Request Body
FeldTypBeschreibung
username *stringAnzeigename des Users
rolestringviewer | editor | admin (default: viewer)
expires_ininteger | nullGültigkeitsdauer in Sekunden. null = unbegrenzt
Beispiel Request
{
  "username":   "alice",
  "role":       "editor",
  "expires_in": 86400
}
Response 201
{
  "token":    "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "username": "alice",
  "role":     "editor"
}
Der JWT-String wird nur einmalig zurückgegeben. Bitte sicher aufbewahren.
GET /api/auth/tokens Alle Tokens auflisten admin ▾

Gibt alle bekannten Tokens zurück (ohne den JWT-String selbst).

Response 200
{
  "tokens": [
    {
      "jti":        "a1b2c3d4e5f6...",
      "username":   "alice",
      "role":       "editor",
      "created_at": 1710000000,
      "expires_in": 86400,
      "revoked":    false
    }
  ]
}
DELETE /api/auth/tokens/{jti} Token widerrufen admin ▾

Sperrt ein Token sofort. Der jti-Wert kommt aus der Token-Liste.

Response 200
{ "revoked": "a1b2c3d4e5f6..." }
GET /api/auth/me Eigenen User abfragen viewer ▾
Response 200
{
  "username":   "alice",
  "role":       "editor",
  "expires_at": 1710086400   // null = kein Ablaufdatum
}

  "filename": "nw-hamburg.png",
  "url":      "/backgrounds/nw-hamburg.png",

Migration

POST /api/migrate NagVis-1-.cfg importieren admin ▾

Nimmt eine NagVis-1-Konfigurationsdatei entgegen, parst alle Objekttypen und legt die Map direkt im MapStore an.

Query-ParamTypDefaultBeschreibung
map_idstring(aus Dateiname)Ziel-Map-ID
canvas_wfloat1200Canvas-Breite in Pixeln für Koordinaten-Umrechnung
canvas_hfloat800Canvas-Höhe in Pixeln
dry_runboolfalseNur Vorschau, nichts speichern
cURL Beispiel
curl -X POST \
  "http://localhost:8080/api/migrate?canvas_w=1200&canvas_h=800&dry_run=false" \
  -H "Authorization: Bearer <admin-token>" \
  -F "file=@datacenter.cfg"
Response 200
{
  "dry_run":      false,
  "map_id":       "datacenter",
  "title":        "Datacenter Hamburg",
  "background":   "images/maps/dc-hamburg.png",
  "object_count": 24,
  "warnings": [
    { "type": "host", "field": "host_name", "message": "host_name fehlt" }
  ],
  "skipped": 1,
  "note": "Hintergrundbild 'images/...' muss manuell hochgeladen werden."
}
Das Hintergrundbild wird nicht automatisch übertragen. Den Pfad aus background notieren und anschließend per POST /api/maps/{id}/background hochladen.

WebSocket

Der Server sendet Diffs – nur geänderte Objekte werden übertragen. Neu verbundene Clients erhalten sofort einen vollständigen Snapshot.

WS /ws/map/{map_id} Live-Updates für eine Map viewer ▾
Verbindung aufbauen
const ws = new WebSocket(
  `ws://localhost:8080/ws/map/datacenter-a?token=${myToken}`
);
Token als Query-Parameter ?token=<jwt>. Fehlendes oder ungültiges Token → Close-Code 4001.
WS /ws/global Alle Events (Admin/Debug) admin ▾

Empfängt alle Events aller Maps. Nur für admin-Tokens. Nützlich für Monitoring, Debugging und Multi-Map-Dashboards.

WebSocket – Events & Kommandos

Server → Browser

snapshot
Vollständiger Status-Snapshot. Wird sofort nach dem Verbindungsaufbau gesendet.
status_update
Geänderte Hosts und Services seit dem letzten Poll. Enthält change_type pro Objekt.
heartbeat
Kein State-Change erkannt. Bestätigt dass der Server noch läuft.
object_added
Ein neues Objekt wurde über die REST-API zur Map hinzugefügt.
object_removed
Ein Objekt wurde gelöscht.
object_updated
Eigenschaften eines Objekts wurden geändert (props-PATCH).
backend_error
Livestatus nicht erreichbar. Enthält message.
pong
Antwort auf ping-Kommando.
status_update – vollständiges Beispiel
{
  "event":            "status_update",
  "ts":               1710000030,
  "poll":             5,
  "elapsed":          43,               // ms
  "hosts": [
    {
      "name":          "srv-web-01",
      "state":         1,
      "state_label":   "DOWN",
      "change_type":   "state_change", // state_change|ack_change|downtime_change|output_change
      "acknowledged":  false,
      "in_downtime":   false,
      "backend_id":    "default"
    }
  ],
  "services":         [],
  "downtime_started": [ "srv-backup-01" ], // optional
  "downtime_ended":   []                   // optional
}

Browser → Server (Kommandos)

Force-Refresh – sofortigen Poll auslösen
ws.send(JSON.stringify({ "cmd": "force_refresh" }));
Ping / Keepalive
ws.send(JSON.stringify({ "cmd": "ping" }));
// → { "event": "pong" }

WebSocket Close-Codes

CodeBedeutung
4001Token fehlt oder ungültig
4003Rolle nicht ausreichend (nur bei /ws/global)

Reconnect-Empfehlung

ws.onclose = (e) => {
  if (e.code === 4001 || e.code === 4003) return; // Auth-Fehler: kein Retry
  setTimeout(() => connect(), 3000); // Netzwerkfehler: Retry nach 3s
};
